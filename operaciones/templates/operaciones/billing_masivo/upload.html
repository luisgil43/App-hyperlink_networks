Important rules

  • Job Code must match exactly. Example: C-123 is not the same as C.123.
  • Only spaces at the beginning or end of a cell are cleaned.
  • Quantity cannot be zero.
  • Direct discounts require negative quantities.
  • Normal billings require positive quantities.
  • Every selected technician must have exact prices for the same Client, City, Project, Office and Job Code.
  • If one row has an error, no billing will be created.

Technician payment mode

In the Excel template, use the column tech_payment_mode. The value must be exactly full or split.

full — Full amount for each technician

Use full when every selected technician must receive their full technical rate. Example: if there are 2 technicians, quantity is 1, and each technician rate is 100, each technician receives 100. Technical total = 200.

split — Split between technicians

Use split when the technical amount must be divided between the selected technicians. Example: if there are 2 technicians, quantity is 1, and the technician rate is 100, each technician receives 50%. Technical total = 100.

Do not write “Full amount”, “Split between technicians”, “yes”, “no”, or any other text in this column. Only full or split are valid.
